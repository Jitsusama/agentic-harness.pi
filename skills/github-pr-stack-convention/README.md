# github-pr-stack-convention

Operational rules for managing stacked PRs: detecting
stacks, verifying base chains, merging in order and
avoiding the pitfalls that auto-close dependent PRs.

Loaded when merging PRs, rebasing branches with
dependents, or creating PRs based on other PRs.

See `github-pr-format` for stack notation in PR
descriptions (`👈 👇 👉`).
